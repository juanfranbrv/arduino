Los pulsadores del kit tienen 4 patas, pero eléctricamente funcionan como 2 conexiones. Dos patas ya están unidas internamente entre sí, y las otras dos también forman otro par. Al pulsar, el botón conecta esos dos lados.

![Infografía del pulsador de 4 patas](/fichas/pulsadores-led/pulsador-4-patas-infografia.png)

<Activity id="act-02" title="Leer un pulsador en el monitor serie" validation="Comprobar que Arduino puede leer un botón y mostrar en el monitor serie un valor estable que cambia al pulsar y soltar.">

Ahora Arduino va a leer el pulsador. Conecta un lado del pulsador al pin `8` y el otro lado a `GND`.

![Pulsador conectado a pin digital y GND](/fichas/pulsadores-led/pulsador-monitor-serie.png)

<KeyConcept title="El monitor serie">
El monitor serie es una ventana del Arduino IDE donde la placa puede escribir mensajes. Es como un chat: Arduino escribe y tu lees en el ordenador.

Para usarlo, abre el monitor serie con el icono de la lupa o desde `Herramientas > Monitor Serie`. Comprueba que abajo aparece `9600 baudios`, el mismo valor que usa `Serial.begin(9600)`.

- `Serial.begin(9600)` abre la comunicación.
- `Serial.print()` escribe sin saltar de línea.
- `Serial.println()` escribe y salta a la línea siguiente.
</KeyConcept>

<KeyConcept title="Una entrada digital necesita un estado definido">
Una entrada digital no debe quedar al aire. Si un pin no está conectado a un estado claro, puede leer ruido y cambiar entre `0` y `1` sin que pulses nada.

`INPUT_PULLUP` activa una resistencia interna que mantiene el pin en `HIGH` cuando el botón está suelto. Al pulsar, conectas el pin a `GND` y la lectura pasa a `LOW`.
</KeyConcept>

<WatchOut>
Con `INPUT_PULLUP`, la lógica queda invertida: `LOW` significa pulsado.
</WatchOut>

```cpp
// Ficha 05 - Actividad 2: leer un pulsador en el monitor serie

const int PIN_BOTON = 8;

void setup() {
  Serial.begin(9600);
  pinMode(PIN_BOTON, INPUT_PULLUP);
}

void loop() {
  int valor = digitalRead(PIN_BOTON);

  Serial.print("Pin 8 = ");
  Serial.println(valor);

  delay(200);
}
```

<Activity id="act-03" title="Encender el LED y dejarlo encendido" validation="Usar una entrada digital para activar una salida y observar que HIGH deja el LED encendido hasta que el programa escriba otro estado.">

Vas a usar el pulsador para encender un LED. La diferencia importante es que el LED se queda encendido.

![Pulsador y LED en protoboard](/fichas/pulsadores-led/un-boton-led.png)

<KeyConcept title="HIGH y LOW son estados">
`HIGH` y `LOW` son estados, no pulsos. Si haces `digitalWrite(PIN_LED, HIGH)`, el pin se queda en `HIGH` hasta que el programa escriba otra cosa.
</KeyConcept>

<WatchOut>
Si nunca escribes `LOW`, el LED no se apaga.
</WatchOut>

```cpp
// Ficha 05 - Actividad 3: pulsa una vez y se queda encendido

const int PIN_BOTON = 8;
const int PIN_LED   = 10;

void setup() {
  pinMode(PIN_BOTON, INPUT_PULLUP);
  pinMode(PIN_LED, OUTPUT);
}

void loop() {
  if (digitalRead(PIN_BOTON) == LOW) {
    digitalWrite(PIN_LED, HIGH);
  }
}
```

Pulsa el botón una vez. El LED se enciende y se queda así porque el programa no contiene ninguna orden para apagarlo.

<Activity id="act-04" title="Encender el LED solo mientras se pulsa" validation="Usar if y else para controlar los dos casos: LED encendido mientras el botón está pulsado y apagado al soltarlo.">

Ahora el LED solo debe estar encendido mientras mantienes pulsado el botón. Para eso necesitas decir qué ocurre en los dos casos: pulsado y suelto.

<KeyConcept title="else define el otro caso">
`else` define que debe ocurrir cuando la condición del `if` no se cumple.
</KeyConcept>

<WatchOut>
La condición sigue siendo `digitalRead(PIN_BOTON) == LOW`, porque el botón usa `INPUT_PULLUP`.
</WatchOut>

```cpp
// Ficha 05 - Actividad 4: LED encendido mientras pulsas

const int PIN_BOTON = 8;
const int PIN_LED   = 10;

void setup() {
  pinMode(PIN_BOTON, INPUT_PULLUP);
  pinMode(PIN_LED, OUTPUT);
}

void loop() {
  if (digitalRead(PIN_BOTON) == LOW) {
    digitalWrite(PIN_LED, HIGH);
  } else {
    digitalWrite(PIN_LED, LOW);
  }
}
```

La diferencia con la actividad anterior está en el `else`: ahora el programa también fuerza el apagado cuando el botón está suelto.

<Activity id="act-05" title="Controlar un LED con dos botones" validation="Separar dos órdenes de control: un botón enciende el LED y otro botón lo apaga.">

Vas a usar dos botones como si fueran mandos separados: uno para encender y otro para apagar.

![Dos pulsadores y un LED](/fichas/pulsadores-led/dos-botones-led.png)

<WatchOut>
Si pulsas los dos botones a la vez, manda el último `if` que se ejecuta en el programa.
</WatchOut>

```cpp
// Ficha 05 - Actividad 5: dos botones ON/OFF

const int BOTON_ON  = 8;
const int BOTON_OFF = 7;
const int PIN_LED   = 10;

void setup() {
  pinMode(BOTON_ON, INPUT_PULLUP);
  pinMode(BOTON_OFF, INPUT_PULLUP);
  pinMode(PIN_LED, OUTPUT);
}

void loop() {
  if (digitalRead(BOTON_ON) == LOW) {
    digitalWrite(PIN_LED, HIGH);
  }

  if (digitalRead(BOTON_OFF) == LOW) {
    digitalWrite(PIN_LED, LOW);
  }
}
```

Prueba qué ocurre si pulsas los dos botones al mismo tiempo.

<Activity id="act-06" title="Crear un botón con memoria" validation="Crear un interruptor con un solo botón detectando el cambio de suelto a pulsado y recordando el estado del LED.">

Un solo botón puede funcionar como interruptor. Para conseguirlo, el programa necesita recordar si el LED está encendido y detectar el instante exacto en que el botón pasa de suelto a pulsado.

![Un pulsador con memoria para alternar el LED](/fichas/pulsadores-led/toggle-led.png)

<KeyConcept title="Detectar el cambio, no solo el estado">
Para hacer un interruptor con memoria, no basta con saber si el botón está pulsado. Hay que detectar el cambio: de `HIGH` a `LOW`.

Una variable `bool` puede recordar si el LED está encendido (`true`) o apagado (`false`).
</KeyConcept>

<WatchOut>
El `loop()` se ejecuta muchas veces por segundo. Si cambias el LED mientras el botón sigue pulsado, el programa puede encenderlo y apagarlo demasiadas veces.
</WatchOut>

```cpp
// Ficha 05 - Actividad 6: un botón con memoria

const int PIN_BOTON = 8;
const int PIN_LED   = 10;

bool ledEncendido   = false;
int estadoAnterior  = HIGH;

void setup() {
  pinMode(PIN_BOTON, INPUT_PULLUP);
  pinMode(PIN_LED, OUTPUT);
}

void loop() {
  int estadoActual = digitalRead(PIN_BOTON);

  if (estadoAnterior == HIGH && estadoActual == LOW) {
    ledEncendido = !ledEncendido;
    digitalWrite(PIN_LED, ledEncendido ? HIGH : LOW);
    delay(50);
  }

  estadoAnterior = estadoActual;
}
```

Detalles importantes:

- `bool` solo puede valer `true` o `false`.
- `!ledEncendido` significa "lo contrario de ledEncendido".
- `estadoAnterior` guarda cómo estaba el botón en la vuelta anterior del `loop()`.
- `delay(50)` ayuda a evitar rebotes del pulsador.
