![Infografía sobre zumbador activo y pasivo](/fichas/zumbador-activo/zumbador-activo-infografia.png)

Físicamente, un zumbador produce sonido porque una pieza interna vibra muy deprisa y empuja el aire que tiene alrededor. En el zumbador activo esa vibración la provoca un pequeño circuito interno: al recibir corriente, genera por sí mismo una frecuencia fija y la convierte en un pitido.

<Activity id="act-01" title="Conectar el zumbador activo a 5V" validation="El zumbador suena de forma continua al alimentar el circuito y deja de sonar al desconectarlo." environment="placa">

Esta primera actividad es completamente eléctrica: no necesitas programa.

Coloca el zumbador activo en la protoboard e identifica sus patas:

- La pata `+` suele ser la larga o estar marcada.
- La pata `-` suele ser la corta.

Haz este montaje:

- pata `+` del zumbador a `5V`
- pata `-` del zumbador a `GND`

Al conectar la placa por USB, el zumbador debe sonar de forma continua.

<KeyConcept title="Activo no significa más fuerte, sino más simple">
El zumbador activo lleva un oscilador interno. No hace falta generar notas ni frecuencias: cuando recibe corriente, suena.
</KeyConcept>

<WatchOut>
Si no suena, revisa la polaridad. En muchos casos el problema es simplemente que el zumbador está al revés.
</WatchOut>

</Activity>

<Activity id="act-02" title="Programar un sonido intermitente en el pin 8" validation="El zumbador alterna sonido y silencio con un ritmo estable controlado por el programa." environment="placa">

Ahora el zumbador ya no va a ir a `5V` directo. Conéctalo así:

- pata `+` del zumbador al pin `8`
- pata `-` del zumbador a `GND`

Sube este programa:

```cpp
const int PIN_BUZZER = 8;

void setup() {
  pinMode(PIN_BUZZER, OUTPUT);
}

void loop() {
  digitalWrite(PIN_BUZZER, HIGH);
  delay(200);
  digitalWrite(PIN_BUZZER, LOW);
  delay(800);
}
```

El zumbador hace un pitido corto y después queda en silencio antes de repetir.

<KeyConcept title="Arduino lo controla igual que un LED">
Para un zumbador activo, `HIGH` significa sonido y `LOW` significa silencio. Se trabaja como una salida digital normal.
</KeyConcept>

Prueba a cambiar los tiempos:

- `delay(200)` y `delay(800)` para un aviso corto
- `delay(500)` y `delay(500)` para un ritmo regular
- `delay(50)` y `delay(50)` para una alarma más rápida

</Activity>

<Activity id="act-03" title="Añadir un LED al mismo circuito" validation="El LED y el zumbador se activan y desactivan a la vez con el mismo ritmo." environment="placa">

Mantén el zumbador en el pin `8` y añade un LED externo:

- ánodo del LED al pin `10` pasando por una resistencia de `220 ohm`
- cátodo del LED a `GND`

Sube este programa:

```cpp
const int PIN_BUZZER = 8;
const int PIN_LED = 10;

void setup() {
  pinMode(PIN_BUZZER, OUTPUT);
  pinMode(PIN_LED, OUTPUT);
}

void loop() {
  digitalWrite(PIN_BUZZER, HIGH);
  digitalWrite(PIN_LED, HIGH);
  delay(200);

  digitalWrite(PIN_BUZZER, LOW);
  digitalWrite(PIN_LED, LOW);
  delay(800);
}
```

Ahora tienes una alarma basica con señal sonora y señal luminosa.

<WatchOut>
El LED necesita resistencia. El zumbador activo no se monta igual que el LED: no copies esa parte del circuito sin revisar el esquema.
</WatchOut>

<OptionalChallenge>
Cambia el programa para que el LED parpadee más rápido que el zumbador. Así verás que dos salidas pueden seguir ritmos distintos dentro del mismo `loop()`.
</OptionalChallenge>

</Activity>

<Activity id="act-04" title="Usar un pulsador para activar zumbador y LED" validation="Mientras el pulsador está presionado, el zumbador suena y el LED se enciende; al soltar, ambos se apagan." environment="placa">

Mantén el montaje de la actividad anterior y añade un pulsador:

- un lado del pulsador al pin `2`
- el otro lado del pulsador a `GND`

En esta actividad vas a usar `INPUT_PULLUP`, así que no necesitas resistencia externa para el botón.

```cpp
const int PIN_BUZZER = 8;
const int PIN_LED = 10;
const int PIN_BOTON = 2;

void setup() {
  pinMode(PIN_BUZZER, OUTPUT);
  pinMode(PIN_LED, OUTPUT);
  pinMode(PIN_BOTON, INPUT_PULLUP);
}

void loop() {
  if (digitalRead(PIN_BOTON) == LOW) {
    digitalWrite(PIN_BUZZER, HIGH);
    digitalWrite(PIN_LED, HIGH);
  } else {
    digitalWrite(PIN_BUZZER, LOW);
    digitalWrite(PIN_LED, LOW);
  }
}
```

<KeyConcept title="Con INPUT_PULLUP la lectura va invertida">
Cuando el botón está suelto, el pin lee `HIGH`. Cuando pulsas y conectas a `GND`, el pin lee `LOW`.
</KeyConcept>

Si todo está bien montado, al pulsar se enciende el LED y el zumbador suena al mismo tiempo.

<OptionalChallenge>
Haz que al pulsar el botón el zumbador no suene continuo, sino con pitidos cortos mientras mantienes el botón apretado.
</OptionalChallenge>

</Activity>
